Frame Melee
Instructions




  • Main Menu
    • Single View: Utilizes entire screen for single character frame parsing.
    • Split View: Splits the screen into two equal windows for double character frame parsing.
    • Stat Rankings: Displays a list of common character properties.
    • Instructions: You are here.
    • Donate: Links to paypal where users who appreciate this app can support it.
    • Home/Logo: Go back to main menu
    • Credits: Gives a special thanks to a certain group of people whom this application could not be done without. A donation link can be found here.
  • Character/Action selection
    • Select one of the 26 available characters in Super Smash Bros Melee.
    • After selecting a character, user is brought to choose an action.
    • Movement has statistics specific for that character
    • Smashes, aerials, basic attacks, and specials can be clicked on to view the action specific frame data.
    • Refresh button: brings all windows back to character selection.
  • Frame Analysis
    • Important: Wait for image load to complete before pressing any of the control buttons to avoid playback issues.
    • White Loading bar: Loading images from server.
    • Grey Loading bar: Parsing images locally.
    • Top left text:
      • Character name: action
      • Current frame state
      • Current frame / Total frames
    • Top right images:
      • Character image: click to view recorded frame data for specified action
      • Back image: button to go back to ability select
      • Control Buttons:
      • Play: Play action on repeat from current frame.
      • Pause: Pause action when on play, freezes on current frame.
      • Restart: Restart action from frame 1.
      • Forwards: Go forward 1 frame.
      • Backwards: Go rewind 1 frame.
      • Progress bar: Green bar that displays the percentage of the current frame in relation to total frames
      • Image of Action
      • Image of character in action with its hitboxes, hurtboxes, state, etc.
      • **Drag image left to rewind frames
      • **Drag image right to fast forward frames
  • Term Clarification
    • Walk Speed: Slow ground movement in which a character can move while still being actionable to smashes and tilts.
    • Run Speed: Fast ground movement after a dash in which a character cannot smash or tilt unless in full stop.
    • Stop Speed: How many frames it would take to be standing still and fully actionable after running.
    • Turn Speed: How many frames it would take to be moving the opposite direction while running.
    • Dash Speed: Burst of speed where a character can change directions in 1 frame.
    • Dash to Run: Number of frames from the start of a dash where the dash becomes a run.
    • Dash Dance Length: Speed of dash in relation to dash frames.
    • Heaviness: How much % it takes to be K.O vertically from the ceiling.
    • Avg Fall Speed: Factor in falling acceleration and top speed to how fast a character falls from the air.
    • Fast Fall Speed: Gains a faster fall speed when a user taps down on the control stick.
    • Short Jump Height: Height where a user release jump at least 2 frames before being in the air.
    • Jump Speed: How many frames a character will be airborne after pressing jump.
    • Aerial Move Speed: The horizontal speed in which a character can move in the air.
    • Wavedash Distance: (Traction) Distance of how far a character's single wavedash is. Wave dashing is a move where a character jumps and airdodges diagonally
    • Grab Range: How far a character lunges for a grab while standing still.
    • Shield Size: How big the shield of a character initially is. Not taking the body:shield ratio into consideration.
    • Dodges, Rolls: Invulnerability frames out of how many frames the entire action takes.
    • Hitbox: When in contact with a hurtbox, the owner of the hurtbox gets 'hit'.
    • Hurtbox: Area of the body in which the owner can be vulnerable to hitboxes.
    • Invulnerable: Cannot be hit.
    • Shot: Projectile comes out.
    • IASA: (Interruptible As Soon As) Can begin a new action during interruptible frames even if the full animation of the previous action is not yet completed. Some interruptible frames may only be interrupted by certain actions.
    • Auto-Cancel: Landing at the start or end of some aerial attacks may produce only 4-5 frames of jump lag, instead of its full landing lag.
